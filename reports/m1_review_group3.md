# Milestone Review

## Project Status

As of **Milestone 1 (March 9, 2026)**, the core functionality of **The Hive Platform** has been implemented and deployed. The system currently includes the web application, backend API, project documentation on the GitHub wiki, and a pre-release version prepared for demonstration.

The baseline service lifecycle is operational, including:
- User registration and login
- Service creation and browsing
- Transactions
- Messaging
- Ratings
- Forum and event features
- Map-based discovery
- Administrative balance adjustments

In addition, a **mobile APK** has been prepared to support testing of the platform on mobile devices.

Looking ahead to **Milestone 2 (April 13, 2026)**, we still have some issues and missing features on the website such as:
- Recommendation feature
- Search bar improvements
- Reporting strategy
- Forum improvements

The mobile version currently supports only some core functionality. Our main focus will be to complete the website side and add more features (which will be planned in detail) to the mobile application. Efforts will also focus on addressing issues identified through broader user testing and improving the overall usability of the platform.

---

## Customer Feedback and Reflections

Based on the milestone demonstration, the customer appeared generally satisfied with the current progress of the platform. One suggestion raised during the discussion was the possibility of allowing users to add pictures to their reviews. Since this feature is not currently supported, we have added it to our future development plan for further evaluation and potential implementation.

---

## Evaluation of Deliverables and Impact on Plan

All deliverables planned for **Milestone 1** have been completed. A small number of issues remained open at the milestone date, but they were resolved within a few days after the deadline.

Currently, several issues—both bug fixes and feature implementations—have already been assigned to **Milestone 2**. A more detailed development plan will be prepared for Milestone 2 based on the current backlog and the feedback received during the milestone review.

In addition, until Milestone 1, we followed a more separated distribution of responsibilities in order to make integration easier during the early development phase. Based on the feedback we received, we plan to involve every team member more actively in both implementation and code review, especially for the new features that will be added in the upcoming milestones. We have already started assigning issues accordingly to support this approach.

---

## Tools and Processes Evaluation

We used **GitHub issues** and **pull requests** to track changes and maintain traceability throughout development. Merges and review discussions were documented in the PR history, which helped us understand how different parts of the system evolved.

The **GitHub wiki** served as a central place for requirements, design artifacts, and meeting notes. In addition, **GitHub Actions** and **Docker Compose** helped us keep development environments consistent and reduced configuration issues. **FastAPI’s autogenerated documentation** was also useful for quickly validating API endpoints during development.

During the first weeks of the project, we did not pay enough attention to code reviews. However, after receiving feedback about this a week before Milestone 1, we started to review pull requests more carefully and follow a more disciplined review process. We plan to continue this approach in the upcoming milestones.

For **Milestone 2**, we also plan to introduce a lightweight QA checklist for each pull request and increase automated test coverage to reduce potential rework.

**Requirements Addressed In This Milestone**

- User registration and login flow.
- User profiles with badges, ratings, and images.
- Service creation (offers/needs), browsing, and basic filtering.
- Transaction lifecycle and timebank balance updates.
- Chat and messaging around services and transactions.
- Forum and events for community interaction.
- Map-based discovery.
- Admin adjustments for timebank balances.
- Content safety via profanity checks and moderation filters.
- Mobile application foundation and service lifecycle coverage.
